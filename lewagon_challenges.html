Basic SQL Notions - Data Transformation

Understanding SQL Basics
SQL (Structured Query Language) is the standard language for managing and manipulating databases. In this challenge, you'll learn the fundamental concepts of SQL and how to write basic queries.

Instructions:

  1. Understand the basic structure of SQL queries
  2. Learn about SELECT, FROM, and WHERE clauses
  3. Practice writing simple queries
  4. Understand the concept of tables and relationships

Key Concepts:

  • SELECT: Specifies which columns to retrieve
  • FROM: Specifies which table to query
  • WHERE: Filters the results based on conditions
  • Basic syntax: SELECT column1, column2 FROM table_name WHERE condition;
-- Example query SELECT first_name, last_name FROM users WHERE age > 18;
Advanced Filtering Techniques
Learn how to filter data using various SQL conditions and operators to get exactly the data you need.

Instructions:

  1. Master comparison operators (=, <, >, <=, >=, !=)
  2. Learn logical operators (AND, OR, NOT)
  3. Understand pattern matching with LIKE
  4. Practice complex filtering conditions

Advanced Filtering:

  • Use IN for multiple possible values
  • Use BETWEEN for range queries
  • Use LIKE with wildcards (% and _)
  • Combine conditions with AND/OR
-- Example of complex filtering SELECT * FROM products WHERE price BETWEEN 10 AND 100 AND category IN ('Electronics', 'Books') AND name LIKE '%phone%';
Data Organization
Learn how to sort and organize your query results to make data more meaningful and easier to analyze.

Instructions:

  1. Master the ORDER BY clause
  2. Understand ascending and descending order
  3. Learn to sort by multiple columns
  4. Practice combining sorting with filtering

Sorting Tips:

  • ASC for ascending order (default)
  • DESC for descending order
  • Multiple columns are sorted in sequence
  • NULL values can be handled with NULLS FIRST/LAST
-- Example of multi-column sorting SELECT * FROM employees WHERE department = 'Sales' ORDER BY salary DESC, hire_date ASC;
Summarizing Data
Learn how to use aggregate functions to summarize and analyze data effectively.

Instructions:

  1. Understand aggregate functions (COUNT, SUM, AVG, MIN, MAX)
  2. Learn to use GROUP BY
  3. Master HAVING clause
  4. Practice combining aggregation with other clauses

Aggregation Concepts:

  • COUNT counts rows or non-null values
  • SUM adds up numeric values
  • AVG calculates the mean
  • GROUP BY creates groups for aggregation
  • HAVING filters after aggregation
-- Example of aggregation SELECT department, COUNT(*) as employee_count, AVG(salary) as avg_salary FROM employees GROUP BY department HAVING COUNT(*) > 5 ORDER BY avg_salary DESC;
Working with Multiple Tables
Master the art of combining data from multiple tables using different types of joins.

Instructions:

  1. Understand different types of joins (INNER, LEFT, RIGHT, FULL)
  2. Learn to use table aliases
  3. Master join conditions
  4. Practice complex joins with multiple tables

Join Types:

  • INNER JOIN: Only matching records
  • LEFT JOIN: All records from left table
  • RIGHT JOIN: All records from right table
  • FULL JOIN: All records from both tables
-- Example of multiple joins SELECT e.name, d.department_name, p.project_name FROM employees e LEFT JOIN departments d ON e.department_id = d.id LEFT JOIN projects p ON e.project_id = p.id WHERE d.location = 'New York';
